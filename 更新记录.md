# AnClaw 更新记录（面向非技术用户）

这份文档用于记录每次版本更新的重点变化，帮助你快速了解：新增了什么、修复了什么、体验提升在哪里。

---

## 1.1.3（当前整理）

### 功能更新
- 聊天入口和本地服务协作流程进一步梳理，启动后进入聊天的路径更稳定。
- 配置迁移能力增强：旧配置升级到新版本时，保留原有可用设置的能力更好，减少重复配置。
- 网关与本地服务联动优化，为后续远程通道和插件能力扩展打好了基础。
- 技能识别路径统一到当前应用数据目录，技能市场安装结果与聊天页状态显示保持一致。

### 修复内容
- 修复了部分场景下“界面显示已运行，但聊天页一直转圈”的问题。
- 修复了网关状态判断不一致导致的误判问题，减少“看起来启动了但实际不可用”的情况。
- 修复了升级和重启过程中的部分配置兼容问题，降低异常中断概率。
- 修复了历史目录与当前目录混用时，技能已安装却被误判为未安装的问题。
- 修复了少数旧检测逻辑仍读取历史目录时，出现“目录空白、脚本缺失”的误判问题。
- 修复了旧版探测读取 `workspace/skills/skills/...` 时无法识别 `desktop-control` 的问题，避免反复提示未安装。
- 修复了聊天窗口上滑查看历史后发送新消息不自动回到底部的问题，避免看不到最新回复。
- 修复了部分模型请求会因 token 预算异常变成负数而直接报错的问题，降低 `max_tokens` 参数错误导致的发送失败。

### 性能优化
- 显著减少插件重复扫描带来的等待时间，连续对话时响应更快。
- 启动前自动清理历史临时文件，避免残留文件拖慢启动和首条消息响应。
- 降低不必要的网络探测和环境探测开销，减少卡顿和长时间等待。
- 减少历史技能目录的冗余扫描，降低状态判定时的额外开销。

### 对你有什么影响
- 打开应用后更容易“马上可用”。
- 聊天等待时间更短，连续使用更顺滑。
- 升级后稳定性更好，遇到异常的概率更低。
- 在技能市场安装成功后，聊天页更不容易再出现“未安装成功”的误提示。
- 即使你的环境里还有旧版检查逻辑，也能更稳定识别 `desktop-control` 已安装状态。
